    自动驾驶综述

    Autonomous Driving Overview — 自动驾驶就是让车在开放道路上自己完成"看路—预判—决策—操作"的闭环;读懂它先要分清自动化等级、四层流水线和传感器套件这三件事.

    难度：Intermediate 前置知识：MPC 后续阅读：自动驾驶范式与方法谱系、协变量偏移与闭环鸿沟

    1. 什么是自动驾驶（任务定义）

    自动驾驶（Autonomous Driving, AD）要解决的，是一个部分可观测、随机、多智能体、安全攸关的序贯决策问题：车辆基于车载传感器对周遭环境的观测，在满足交通规则、舒适性和安全约束的前提下，连续地输出转向、油门、刹车，把自己从起点安全送到终点。

    形式化地，把它看成一个 POMDP（部分可观测马尔可夫决策过程）\(\langle \mathcal{S}, \mathcal{A}, \mathcal{O}, T, Z, R \rangle\)：

    • 状态 \(s_t \in \mathcal{S}\)：本车位姿、速度，以及所有其他交通参与者的位置/意图、道路拓扑、信号灯——其中大部分不可直接观测。
    • 观测 \(o_t \in \mathcal{O}\)：相机像素、LiDAR 点云、radar 回波、IMU/GNSS 读数——带噪声、有遮挡、部分缺失。
    • 动作 \(a_t \in \mathcal{A}\)：底盘控制量（方向盘转角 \(\delta\)、纵向加速度 \(a\)），或上一层的目标轨迹。
    • 目标：最大化期望回报 \(\mathbb{E}\!\left[\sum_t \gamma^t R(s_t, a_t)\right]\)，奖励里编码"到达 + 守规 + 不碰撞 + 平顺"。

    这个问题难在三处：(1) 长尾——99.9% 的里程平淡无奇，事故却发生在极罕见的 corner case；(2) 闭环耦合——车的动作会改变它下一时刻看到的世界，离线指标好不等于上路好（见 协变量偏移与闭环鸿沟）；(3) 安全攸关——失败的代价是人命，容错率远低于推荐系统或聊天机器人。

    2. SAE L0–L5 自动化分级

    业界通用的分级来自 SAE J3016 标准。核心分界线只有一条：谁负责监控驾驶环境、谁在失效时兜底。L0–L2 是"驾驶员开车、系统帮忙"，人始终是责任主体；L3 起系统在其设计域内成为驾驶主体；L4/L5 在域内无需人接管。

    等级 名称 谁监控环境 失效兜底 典型功能 / 代表
    L0 无自动化 驾驶员 驾驶员 AEB 紧急制动、盲区预警（仅预警/瞬时介入）
    L1 驾驶辅助 驾驶员 驾驶员 ACC 自适应巡航 或 车道保持（二选一，单轴）
    L2 部分自动化 驾驶员 驾驶员 ACC + 车道居中同时（横纵双控）；特斯拉 Autopilot/FSD(Supervised)、各家 NOA / 高速领航
    L3 有条件自动化 系统（域内） 驾驶员（需在数秒内接管） 域内可脱手脱眼；Mercedes Drive Pilot、Honda Legend
    L4 高度自动化 系统（域内） 系统（域内自行 MRC 最小风险状态） 域内无人；Waymo One、Cruise（已停）、百度 Apollo Go
    L5 完全自动化 系统（任意域） 系统 任意道路无人——目前不存在量产

    L2 辅助 vs L4 无人，本质区别不在"能力强弱"而在"责任与兜底"。 L2 系统再聪明，法律上人仍须全程监控并随时接管——它是"高级巡航"。L4 在其运营设计域（ODD）内，系统自己负责安全，失效时自己执行最小风险策略（靠边停车），车里可以没有安全员。L3 是尴尬的过渡态：人可以脱手，但被叫醒后要在几秒内接管，"接管交接"本身是个公认的人因难题，所以很多公司选择跳过 L3 直奔 L4。

    3. 感知 → 预测 → 规划 → 控制：四层流水线

    经典自动驾驶栈把整个任务解耦成四个串联模块，相邻模块之间用人为定义的中间表示衔接。这是理解整个领域的骨架。

    Figure 1 — 自动驾驶四层流水线：传感器原始数据经感知抽象成检测框/BEV，预测给出他车未来轨迹，规划求出本车目标轨迹，控制把轨迹跟踪成方向盘/油门/刹车。箭头上标注的"中间表示"是各模块间的人为接口，也是模块化栈最大的优点（可解释）与代价（信息瓶颈）所在。

    3.1 感知（Perception）

    把原始传感器数据变成结构化的世界描述：3D 目标检测与跟踪、车道线/可行驶区域分割、占据栅格（occupancy）、本车定位（SLAM / 高精地图匹配）。现代主流是把多相机/多模态特征投到统一的鸟瞰视角（BEV）空间——LSS（Lift-Splat-Shoot）做几何提升、BEVFormer 用时空注意力——再在 BEV 上做检测与分割；LiDAR 侧则有 VoxelNet / PointPillars / CenterPoint。

    3.2 预测（Prediction / Motion Forecasting）

    给出周围交通参与者未来 3–8 秒的多模态轨迹分布（一个车在路口可能直行也可能右转，必须输出多假设并配概率）。代表方法把地图与历史轨迹编成矢量图（VectorNet、LaneGCN），或用 CVAE 输出多模态（Trajectron++）。

    3.3 规划（Planning）

    在预测出的动态环境里，为本车求一条安全、合规、舒适的目标轨迹。常见做法：行为决策（变道/让行的有限状态机）+ 轨迹优化（在 Frenet 坐标系下采样格点或求解带约束优化），把碰撞、舒适、进度编码成代价函数。

    3.4 控制（Control）

    把规划出的轨迹跟踪成底盘信号。横向常用 Pure Pursuit、Stanley 控制器，或 MPC 在车辆动力学模型上滚动优化；纵向用 PID/MPC 跟速。控制运行频率最高（50–100 Hz），对实时性和稳定性要求最严。

    4. 传感器套件与融合

    没有任何单一传感器能覆盖所有工况，量产与 Robotaxi 都靠互补冗余的多传感器套件。各传感器优劣大致如下：

    传感器 测什么 优势 短板
    相机 颜色/纹理/语义（被动光） 分辨率高、能读交通灯/标志/车道线、便宜 无直接测距、强逆光/夜间/雨雾差、需算力还原 3D
    LiDAR 精确 3D 点云（主动激光） 直接、稠密、精准的距离与几何，夜间不受影响 贵、雨雪雾衰减、点云无语义、运动畸变
    radar（毫米波） 距离 + 径向速度（多普勒） 直接测速、穿雨雾烟尘、远距、廉价、全天候 角分辨率低、易多径杂波、难分辨静止物
    IMU 角速度 + 加速度 高频（数百 Hz）、不依赖外部、短期精准 误差随时间积分漂移，须被其他源校正
    GNSS（GPS 等） 全局绝对位置 提供全局参考、长期不漂移 低频、城市峡谷/隧道失锁、米级精度（需 RTK/IMU 融合）

    互补关系清晰：相机给语义、LiDAR/radar 给几何与速度、IMU+GNSS 给定位；radar 全天候补相机/LiDAR 的恶劣天气短板。一个经典争论是纯视觉（特斯拉，靠相机 + 神经网络估深度，省成本）vs 多模态（Waymo 等，相机 + LiDAR + radar，靠物理测距换冗余安全）。

    4.1 前融合 vs 后融合

    • 后融合（late / object-level）：每个传感器各自跑完感知，输出各自的目标列表，最后在目标级做关联与卡尔曼滤波融合。模块解耦、易工程化、单源故障可隔离，但早期把原始信息丢掉了，弱目标可能在单源里就被滤没。
    • 前融合（early / feature-level）：在原始数据或特征层就把多模态对齐融合（如把 LiDAR 点投到图像、或在 BEV 空间拼接相机特征与点云特征），让网络联合利用全部信息。信息利用最充分、对弱目标更敏感，但对标定/时间同步要求极高，且更像黑箱。现代 BEV 方案（BEVFusion 类）多走前融合/特征级融合。

    两者的取舍与"模块化 vs 端到端"同构：后融合像模块化（接口清晰、可解释），前融合像端到端（联合优化、信息无瓶颈）。

    5. 模块化 vs 端到端 的取舍

    §3 的四层流水线是模块化范式的代名词：各模块用人为中间表示衔接，可解释、可分工、可单独验证，代价是中间表示是信息瓶颈、各模块局部最优不等于全局最优。端到端（E2E）则把整条链路压成一个可微网络，从传感器直接出控制/轨迹，联合优化、无人为瓶颈，但黑箱难验证、数据饥渴，并把模仿学习的协变量偏移暴露得最彻底。

    关键澄清："端到端" ≠ "没有模块"——UniAD（CVPR 2023 Best Paper）保留了感知/预测/规划模块语义，但让梯度从规划目标一路反传到感知，是"端到端可微的模块化"；架构其实是一条连续光谱而非二选一。这条轴、以及它与"学习方法（规则/IL/RL/世界模型）"那条正交轴如何交叉，详见 自动驾驶范式与方法谱系。

    6. Robotaxi vs 量产辅助驾驶

    两条产业路线的目标、约束、商业模式截然不同：

    维度 Robotaxi（L4） 量产辅助驾驶（L2/L2+）
    目标等级 L4，车内无安全员 L2/L2+，人始终监控
    责任主体 运营商/系统 驾驶员
    ODD 圈定城区，高精地图 + 远程监控 高速/城区领航，但人兜底
    传感器 重套件（多 LiDAR + 相机 + radar），成本可摊到运营 成本敏感，趋向纯视觉/轻 LiDAR
    安全冗余 硬件级冗余（双电源/双转向/双计算）+ MRC 以人为最终冗余
    商业模式 按里程收费的出行服务 卖车 / 软件订阅
    代表 Waymo、百度 Apollo Go、文远/小马 特斯拉 FSD(Supervised)、各家高速/城市 NOA

    运营设计域（ODD） 是 L4 的命门：它明确界定系统"在什么条件下保证能用"——地理围栏、天气、时段、道路类型、车速上限。L4 不是"什么都会"，而是"在 ODD 内对自己负责，超出 ODD 就拒绝接管任务或执行 MRC"。

    安全冗余：L4 因为没有人兜底，必须做到故障可运行（fail-operational）——任一关键部件失效后系统仍能把车安全停到最小风险状态（MRC），所以要双路供电、双转向、双计算、多传感器交叉校验。L2 则把驾驶员当作最终冗余，可以做到"故障安全（fail-safe，报警并退出）"即可。这条冗余等级的差异，是 Robotaxi 整车成本远高于量产车的根因之一。

    7. 简史

    • 2004–2007 · DARPA 挑战赛：2004 年首届 Grand Challenge（莫哈维沙漠 240 km）无一车完赛；2005 年 Stanford 的 "Stanley"（Thrun 团队）夺冠，其 Stanley 横向控制器与概率建图成为经典；2007 年 Urban Challenge 把舞台搬到城市路网（含交通规则与他车交互），CMU "Boss" 夺冠。这三场比赛奠定了现代模块化栈的雏形，也培养了日后 Waymo/Cruise/Aurora 的核心人才。
    • 2009– · Waymo（前 Google 自动驾驶项目）：从 DARPA 班底起步，长期押注高精地图 + 多 LiDAR + 模块化栈，2020 年在凤凰城推出无安全员的 Waymo One，是 L4 Robotaxi 的标杆。
    • 2014– · 特斯拉 Autopilot → FSD：从 Mobileye 视觉 ACC/LKA 起步，转向纯视觉 + 影子模式众包数据的量产 L2 路线；2023–2024 年 FSD v12 起把规划/控制大幅端到端化（"photon in, control out"），代表了从模块化栈向端到端学习的产业转向。
    • 2020s · 学术与产业的端到端转向：UniAD（2023 CVPR Best Paper）把端到端可微模块化推向主流，VAD 做矢量化高效化，Wayve 的 MILE/GAIA-1 引入世界模型，nuPlan/CARLA 把闭环评测立为标准——钟摆从 1980s 的端到端（ALVINN）摆到 2010s 的模块化，又在 2020s 用"可微模块化 + 世界模型"把两者融合。

    Common Pitfalls

    1. 把 SAE 等级当能力分而非责任分：L2 与 L4 的真正分界是"谁兜底"，不是"谁更聪明"——一个性能很强的 L2 在法律上仍要求人全程监控。
    2. 以为 L5 已经存在：目前没有任何量产 L5；连 L4 也只在圈定 ODD 内运行。
    3. 混淆"端到端"与"没有模块"：UniAD 反例——端到端只看梯度是否贯通，不看有没有模块（见 §5）。
    4. 忽视 ODD：评判一套 L4 系统不能脱离它的运营设计域；超出 ODD 的"失败"不是 bug 而是设计边界。
    5. 只看离线指标选系统：开环检测/预测 mAP 高 ≠ 闭环上路安全，闭环耦合与协变量偏移会反咬一口。
    6. 过度神化纯视觉或纯 LiDAR：传感器是工程权衡（成本 vs 冗余），不是信仰；多模态前融合靠的是严苛的标定与时间同步。

    前沿趋势

    1. 端到端可微化成主流：UniAD 之后，保留可解释模块语义但梯度贯通的架构成为研究与产业的共同方向。
    2. 占据栅格（occupancy）与世界模型：用通用的 3D 占据/流场表征替代"只检测已知类别"，并用生成式世界模型（GAIA-1）合成稀有危险场景以补长尾。
    3. VLA / 大模型进入驾驶：视觉-语言-动作模型带来语言级可解释的高层决策与对未见场景的常识泛化。
    4. 闭环仿真评测标准化：nuPlan、CARLA Leaderboard 把闭环成功率/违规分作为正式指标，取代单看开环 mAP。
    5. 量产端到端落地：特斯拉 FSD v12 起的端到端转向，正推动行业把"海量真实驾驶数据 + 端到端学习"作为可扩展的产品路线。
