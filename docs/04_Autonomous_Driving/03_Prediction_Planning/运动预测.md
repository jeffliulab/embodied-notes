# 运动预测

> *Motion Prediction — 给定他车的历史轨迹和高精地图,预测它未来几秒可能怎么走;难点不在"预测一条线",而在"未来本来就是多模态的".*
>
> **难度**：Intermediate
> **前置知识**：[自动驾驶范式与方法谱系](../01_Foundations/自动驾驶范式与方法谱系.md)
> **后续阅读**：[规划与决策](规划与决策.md)

---

## 1. 为什么要预测他车（规划需要未来）

规划本质是在**未来**做决策：本车要不要并线、要不要在路口让行、跟车距离留多少，全都取决于周围车、行人、自行车**接下来**会怎么动。如果只有当下的快照（位置 + 速度），规划器要么把所有人当成匀速直线运动（恒速模型，constant-velocity），要么过度保守地把所有可能性都当作障碍——前者会在路口被左转车撞，后者会"冻结"在原地寸步难行（the **freezing robot problem**）。

所以预测是**感知与规划之间的桥**：感知告诉你"现在场上有谁、在哪、朝哪开"，预测把这些状态外推成"未来 3–8 秒每个智能体的可能轨迹分布"，规划再在这个未来分布上做安全且高效的决策。

预测之所以是独立的难题，有三个本质原因：

- **多智能体（multi-agent）**：场上可能有几十个交通参与者，他们的意图互相耦合（你让我先走，我才敢走）。
- **多模态（multi-modal）**：同一辆车在同一时刻，可能直行也可能右转——这是**两个相互排斥的离散意图**，不是同一条轨迹上的噪声。
- **强地图约束**：车几乎一定沿车道行驶,可行的未来被路网拓扑严格约束。把地图喂进模型,是现代预测器相对早期 LSTM 外推的最大增益来源。

## 2. 问题形式化

设场上有 $N$ 个智能体。第 $i$ 个智能体在过去 $T$ 步的状态序列为

$$
\mathbf{s}^{(i)}_{-T:0} = \big(s^{(i)}_{-T}, \dots, s^{(i)}_{0}\big),\quad s^{(i)}_t \in \mathbb{R}^d
$$

其中 $d$ 通常含位置、朝向、速度等。再给定场景上下文 $\mathcal{C}$（高精地图的车道折线、停止线、红绿灯状态、所有智能体的相对关系）。预测目标是估计未来 $H$ 步轨迹的**条件分布**

$$
p\big(\mathbf{Y}^{(i)}_{1:H} \mid \mathbf{s}^{(1:N)}_{-T:0},\, \mathcal{C}\big),\qquad
\mathbf{Y}^{(i)}_{1:H} = (y^{(i)}_1, \dots, y^{(i)}_H),\; y^{(i)}_h \in \mathbb{R}^2 .
$$

关键是输出的是**分布**而不是单条轨迹。实际系统常把它离散成 $K$ 个模态（modes），每个模态给一条轨迹 $\hat{Y}^{(i,k)}$ 和一个概率 $\pi^{(i,k)}$：

$$
p(\mathbf{Y}^{(i)}) \approx \sum_{k=1}^{K} \pi^{(i,k)}\, \mathcal{N}\big(\mathbf{Y}^{(i)} \mid \hat{Y}^{(i,k)}, \Sigma^{(i,k)}\big),\qquad \sum_k \pi^{(i,k)} = 1 .
$$

典型设置：采样频率 $10\,\text{Hz}$，预测时长 $H$ 对应 $3\text{s}$（Argoverse）到 $6\text{s}$（Waymo / nuScenes），模态数 $K=6$（Argoverse / Waymo 榜单的官方设定）。

> **联合预测 vs 边缘预测**：上式对每个 $i$ **独立**输出分布，是 *marginal*（边缘）预测——它会产生"两辆车都预测自己会占据同一个间隙"这种互相矛盾的组合。*Joint*（联合）预测直接建模 $p(\mathbf{Y}^{(1:N)})$，保证场景级一致性，是更难但对规划更有用的形式（Waymo 的 Interaction / Sim Agents 赛道考的就是它）。

## 3. 场景表征

把"历史轨迹 + 地图 + 多智能体关系"编码进网络，有两条主流路线。

### 3.1 栅格化 BEV（rasterized）

把场景渲染成多通道的鸟瞰图（BEV）位图：一个通道画车道，一个画人行横道，几个通道用"运动历史叠帧"画各智能体的过去位置，再用 CNN 提特征。代表是早期的 ChauffeurNet、IntentNet、MultiPath。

- **优点**：实现简单，能直接复用成熟的 CNN；不同要素的空间关系天然对齐。
- **缺点**：分辨率与感受野是死结——栅格太粗丢细节，太细则算力爆炸；车道这种**稀疏、矢量、有方向**的结构被强行塞进稠密像素，极其浪费；远距离的拓扑关系要靠很深的卷积才能传递。

### 3.2 矢量化（vectorized）：VectorNet / LaneGCN

把地图和轨迹都表示成**折线（polyline）**：一条车道中心线是一串有向线段，一条历史轨迹也是一串线段。每个折线先用一个子图（subgraph）聚合成节点特征，再用**全局图网络**让所有折线（车道、智能体）互相交互。这就是 **VectorNet**（Gao et al. 2020）的核心思想——*"用图表示，而不是用像素"*。

**LaneGCN**（Liang et al. 2020）更进一步，专门为车道图建模四类连接：

- **AL / LA**（agent↔lane）：智能体与附近车道的关联；
- **LL**（lane↔lane）：车道之间的拓扑（前驱/后继/左右相邻），用图卷积沿车道网络传播；
- **AA**（agent↔agent）：智能体之间的交互。

矢量化的好处：精确（不丢几何）、稀疏（只算真实存在的元素）、自带方向与拓扑语义。代价是工程更复杂、对地图质量更敏感。今天榜单上的主流方法（LaneGCN、TNT、DenseTNT、HiVT、Wayformer、MTR）几乎全是矢量化 + Transformer/GNN。

| | 栅格化 BEV | 矢量化折线 |
|---|---|---|
| 几何精度 | 受分辨率限制 | 精确 |
| 算力随场景增长 | $\propto$ 像素面积 | $\propto$ 元素数（稀疏） |
| 编码器 | CNN | GNN / Transformer |
| 地图拓扑 | 隐式（靠卷积传） | 显式（车道图边） |
| 代表 | ChauffeurNet, MultiPath | VectorNet, LaneGCN |

## 4. 多模态与不确定性

### 4.1 为什么单一轨迹不够

未来的不确定性有两种本质不同的来源：

- **偶然不确定性（aleatoric）/ 模态内**：同一意图下的连续噪声——同样是直行，速度可快可慢，横向有抖动。可用每个时刻的高斯协方差 $\Sigma_h$ 描述。
- **认知/意图不确定性 / 模态间**：离散的、相互排斥的意图——直行 vs 右转。这**不是**一个高斯能描述的：两条轨迹的"平均"落在马路牙子上，是物理上不可能的路径。

这正是回归单条轨迹（最小化 MSE）的致命伤：MSE 的最优解是条件期望 $\mathbb{E}[Y]$，在双峰分布上等于**两个峰的平均**——一条谁都不像、还可能撞墙的"骑墙"轨迹（mode averaging / mode collapse）。所以预测必须输出**多个模态**。

### 4.2 混合密度网络（MDN）

直接让网络输出 $K$ 组 $(\pi_k, \mu_k, \Sigma_k)$，用负对数似然训练。问题是**模态崩塌**：所有模态会塌到一起、或某些模态永远拿不到梯度。需要 *winner-takes-all*（只对最接近真值的那条模态回传轨迹损失）等技巧稳住训练。

### 4.3 锚点 / 目标条件：MultiPath → TNT → DenseTNT

把"预测一条曲线"拆成**先选目标点、再回归到目标的路径**，让多模态有了清晰的离散载体：

- **MultiPath**：预定义一组轨迹锚点（anchors，对训练集聚类得到），网络只需为每个锚点输出概率 + 残差偏移——把回归变成"分类 + 微调"，天然多模态。
- **TNT**（Target-driveN Trajectory）：三阶段——① 在车道上**采样候选目标点**并打分，② 给每个高分目标回归一条轨迹，③ 对轨迹集合做打分与去重（NMS）。**目标条件**的好处是目标点天然落在车道上，物理可行。
- **DenseTNT**：去掉对稀疏锚点的依赖，直接在**稠密的目标候选点**上做概率热力图预测，再用一个可微的离线优化来选出最终目标集，避免了人工锚点的偏差。

### 4.4 生成式：Trajectron++ / CVAE

**Trajectron++**（Salzmann et al. 2020）用 CVAE：引入离散隐变量 $z$ 表示"高层意图模态"，解码器以 $z$ 为条件生成轨迹。采样不同 $z$ 即得到不同模态，且能优雅地融合地图、智能体异质性（车/人/自行车）与动力学约束。生成式的好处是天然给出样本分布；代价是需要采样足够多才能覆盖所有模态。

## 5. 评测指标

预测的核心难点是"多模态对、还是单模态对"，所以指标几乎都是 **top-$K$** 形式——允许模型给 $K$ 个猜测，只惩罚最好的那个，避免逼模型在双峰上取平均。

设真值未来为 $Y$，模型输出 $K$ 条候选 $\{\hat Y^{(k)}\}$：

- **minADE$_K$**（minimum Average Displacement Error）：取 $K$ 条里平均位移误差最小的那条
$$
\text{minADE}_K = \min_{k}\; \frac{1}{H}\sum_{h=1}^{H} \big\lVert \hat y^{(k)}_h - y_h \big\rVert_2 .
$$
- **minFDE$_K$**（minimum Final Displacement Error）：只看终点
$$
\text{minFDE}_K = \min_{k}\; \big\lVert \hat y^{(k)}_H - y_H \big\rVert_2 .
$$
- **Miss Rate (MR)**：终点误差超过阈值（常用 $2\,\text{m}$）的样本占比——衡量"是否至少有一条候选靠谱"。
- **brier-minFDE**：$\text{minFDE} + (1-\hat\pi^{(k^\*)})^2$，把选中模态的概率好坏也惩罚进去（Argoverse 的官方排序指标）——光"猜中了"不够，还得给对概率。

> **指标陷阱**：单看 minADE$_K$ 会鼓励"撒网"——输出 6 条发散的轨迹总能蒙中一条，却没有任何一条概率可信。这就是为什么 Argoverse 用 brier-minFDE、Waymo 用考虑概率的 mAP（按置信度排序做 AP）作为主榜指标。

| 指标 | 看什么 | 是否惩罚概率 |
|---|---|---|
| minADE$_K$ | 全程平均误差（最优模态） | 否 |
| minFDE$_K$ | 终点误差（最优模态） | 否 |
| Miss Rate | 是否有候选落在阈值内 | 否 |
| brier-minFDE | 终点误差 + 概率校准 | 是 |
| mAP (Waymo) | 按置信度排序的精确率 | 是 |

## 6. 与规划的耦合

### 6.1 朴素串联的两个病

最简单的做法是"预测一次 → 把预测轨迹当固定障碍 → 规划绕开"。它有两个结构性缺陷：

1. **忽视交互（non-reactive）**：预测器假设他车不受本车影响。但现实是他车会**对本车反应**——本车一加塞，对方就会减速让行。把他车当成"录像回放"会导致两种极端：要么过度保守（认定对方不会让，于是永远等），要么误判（用没考虑本车存在的预测去规划）。
2. **冻结机器人问题**：在拥挤场景，若把每个模态都当作不可侵犯的障碍，自由空间被多模态的"未来锥"吃光，规划器找不到可行解而**停滞**——明明人类司机能轻松通过。

### 6.2 预测-规划一体化与博弈式交互

更现代的视角把预测和规划当成一个**耦合的、博弈式**的问题：

- **条件预测 / 本车条件（ego-conditioned）**：预测器输入里加入本车的候选计划，输出 $p(\mathbf{Y}^{\text{他车}} \mid \text{本车计划})$。规划遍历自己的若干候选计划，看哪个引出的他车反应最有利——这就把"反应性"建模进来了。
- **联合预测**：直接输出场景级一致的多智能体联合轨迹（§2），规划在联合分布上评估。
- **博弈论视角**：路口的让行本质是一个**多智能体博弈**，可用纳什/Stackelberg 均衡或层级推理（level-$k$）建模"我猜你会让，所以我走；你猜我会走，所以你让"。
- **可微一体化**：UniAD / VAD 把预测与规划放进同一个可微网络，用最终的规划损失反传去塑造预测——预测不再为"预测精度"优化，而是为"让规划开得好"优化（planning-oriented）。

> 一句话：**预测的终极用户是规划器，预测的好坏最终要用闭环驾驶质量来衡量**，而不是离线的 minADE。一个 minADE 更低但概率乱给、不考虑交互的预测器，可能让规划更糟。

## 方法对比

| 方法 | 表征 | 多模态机制 | 交互建模 | 代表指标取向 |
|---|---|---|---|---|
| 恒速 / 卡尔曼 | 状态向量 | 无（单模态） | 无 | 基线 |
| Social-LSTM | 轨迹序列 | 弱（采样） | social pooling | 人群轨迹 |
| ChauffeurNet / MultiPath | 栅格 BEV | 锚点分类 | 隐式（栅格） | mid-to-mid |
| VectorNet | 矢量折线 + 全局图 | MDN / 多头 | 折线图交互 | minADE/FDE |
| LaneGCN | 车道图 + 智能体图 | 多头回归 | AL/LL/AA 显式 | minADE/FDE |
| TNT / DenseTNT | 矢量 + 目标条件 | 目标采样 → 回归 | 折线图 | minFDE/MR |
| Trajectron++ | 图 + CVAE | 隐变量采样 | 异质图 + 动力学 | KDE-NLL / minADE |
| MTR / Wayformer | 矢量 + Transformer | 可学习意图 query | 注意力 | mAP / minADE |
| UniAD / VAD | 端到端可微 | query 多模态 | 与规划联合 | 闭环 + 规划损失 |

## Common Pitfalls

1. **用 MSE 回归单条轨迹**：在多模态分布上会塌成"骑墙均值"，落到不可行区域（§4.1）。必须用多模态损失（winner-takes-all / 锚点分类 / CVAE）。
2. **只看 minADE$_K$ 选模型**：等于鼓励"撒网式"乱猜，概率不可信。用 brier-minFDE / mAP 这种带概率校准的指标。
3. **把边缘预测当联合预测用**：对每个智能体独立采样会产生互相穿模、抢同一间隙的不一致组合（§2）。
4. **不用地图、纯轨迹外推**：丢掉车道拓扑这一最强先验，远不如矢量化 + 车道图。
5. **non-reactive 串联进规划**：把他车预测当录像回放，导致过度保守或冻结机器人（§6）。
6. **训练/推理的坐标系不一致**：预测对每个智能体常做"以自身为中心 + 朝向对齐"的归一化，忘了在融合/解码时变换回全局系会引入系统性偏差。
7. **离线指标好就上车**：预测的真正考核是闭环驾驶质量，而非开环 minADE。

## 前沿趋势

1. **联合 / 场景级预测**：从 marginal 走向 joint，保证多智能体一致性（Waymo Sim Agents、Scene Transformer、MTR++）。
2. **预测-规划一体化（planning-oriented）**：UniAD / VAD 用规划目标反传塑造预测，详见[规划与决策](规划与决策.md)。
3. **占据流（occupancy flow）预测**：不预测离散轨迹，而预测时空占据 + 流场，天然处理任意多智能体与不确定性（Waymo Occupancy and Flow 赛道）。
4. **基于 Transformer 的统一架构**：Wayformer、MTR 用注意力统一融合地图与智能体，意图用可学习 query 表达，几乎横扫各大榜单。
5. **大模型 / 语言进入预测**：用 LLM/VLM 做意图推理与可解释预测，把"它在等行人，所以会停"这类常识引入。
6. **闭环、反应式评测**：从开环 minADE 转向闭环、可交互的仿真评测（nuPlan、Waymo Sim Agents），逼预测器学会反应性。
