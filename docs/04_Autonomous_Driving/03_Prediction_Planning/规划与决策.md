# 规划与决策 — 从行为决策到运动轨迹

> *Planning & Decision-Making — 自动驾驶把"我该怎么开"拆成两层：**先做离散的行为决策（变道？让行？超车？）**，再求一条满足动力学、舒适、安全约束的**连续运动轨迹**，最后交给控制器跟踪。这一节讲清两层的边界、方法谱系（FSM/POMDP、采样/优化/MPC）、代价函数设计与 RSS 式可验证安全。*

## 1. 规划的两层（行为决策 behavior + 运动规划 motion）

自动驾驶的"决策-规划"（decision & planning）模块位于**预测之后、控制之前**。它接收：自车状态、地图与可行驶区域、感知/[预测](运动预测.md)给出的他车未来轨迹分布；输出一条带时间戳的**参考轨迹** $\{(x_k, y_k, \theta_k, v_k, t_k)\}$ 交给 [车辆控制](../04_Control/车辆控制.md) 跟踪。

绝大多数量产栈把这件事拆成两层，因为它们的数学性质截然不同：

| 层 | 决策对象 | 数学性质 | 时间尺度 | 典型方法 |
|---|---|---|---|---|
| **行为决策 (behavior)** | 离散语义动作：跟车 / 变道 / 让行 / 路口通过 | 离散、组合、含博弈 | 1–8 s | FSM、规则、POMDP、搜索 |
| **运动规划 (motion)** | 连续轨迹 $\xi(t)$ | 连续、带约束的最优化 | 0.1–5 s | 采样、格搜、优化、MPC |

**为什么要分层？** 直接在连续空间里搜索"是否变道"会让优化问题变成强非凸、多模态（每个离散选择对应一个吸引盆）。行为层先**选定一个语义意图**（如"向左变道并插入到 A 车之后"），就把多模态问题坍缩成一个**单模态、近似凸**的轨迹优化，运动层因此能稳定快速求解。代价是分层会丢信息（上层选错下层无法挽救），于是出现 **behavior+motion 联合**（如 lattice 同时枚举行为与轨迹）和**端到端**两条弥合路线。

下图给出这条分层流水线的"视觉 DNA"——离散意图集 → 选中一支 → 连续轨迹束 → 控制跟踪。

## 2. 行为决策（FSM、规则、POMDP 下的不确定决策）

### 2.1 有限状态机 (FSM)

最经典也最易上线的行为决策器。把驾驶语义状态枚举为节点（`LANE_KEEP`、`LANE_CHANGE_LEFT`、`STOP_FOR_LIGHT`、`YIELD`、`OVERTAKE`…），用守卫条件（guard）定义转移。DARPA Urban Challenge 时代的 Junior/Boss、早期 Apollo 的 `scenario` 调度本质都是分层 FSM。

- **优点**：可读、可审计、行为可预测、易写测试。
- **缺点**：状态/转移随场景组合**指数膨胀**（"无保护左转 + 行人 + 自行车 + 雨天"），守卫条件互相打架；难以表达连续的不确定性。

工程上常退化为**分层 FSM + 行为树 (behavior tree)** 控制组合复杂度。

### 2.2 规则 / 成本表 (cost-table) 决策

不显式建状态机，而是**枚举候选语义动作**（如对每条车道生成"保持/左变/右变"），对每个动作算一个标量代价（考虑通行效率、与他车冲突、变道收益），取 argmin。这是 lattice 规划器在行为层的雏形，也是把行为决策"软化"为优化的过渡形态。

### 2.3 POMDP — 不确定下的决策

真实驾驶里他车意图**不可观测**：旁车是要并线还是只是飘了？行人会不会突然横穿？这天然是部分可观马尔可夫决策过程 (POMDP)。

$$
\langle \mathcal{S}, \mathcal{A}, \mathcal{T}, \mathcal{R}, \Omega, \mathcal{O}, \gamma \rangle, \qquad
b'(s') \propto \mathcal{O}(o\mid s',a)\sum_{s} \mathcal{T}(s'\mid s,a)\, b(s)
$$

智能体不在状态 $s$ 上、而在**信念** $b$（状态分布）上决策，目标是最大化期望折扣回报 $\mathbb{E}[\sum_t \gamma^t \mathcal{R}]$。其威力在于自动权衡**信息收集 vs 任务推进**——例如在视线受阻路口先缓行"探头"以缩减信念熵，再决定通过，这正是人类司机的行为。

精确求解 POMDP 是 PSPACE-hard，实践用近似：

- **QMDP / FIB**：忽略未来观测，把信念上的决策退化为对 MDP 解的加权。
- **DESPOT / POMCP**：基于蒙特卡洛信念树搜索，在线展开有限场景（determinized sampling），是无保护左转、汇流等交互场景的主流近似器。
- **多策略法 (MPDM / EUDM)**：枚举少量"宏策略"（自车策略 × 他车意图假设），前向仿真打分，是 lattice/FSM 与 POMDP 的折中，工程友好。

> 与 [运动预测](运动预测.md) 的关系：预测给出 $\mathcal{T}$ 与 $\mathcal{O}$（他车未来分布），决策在其上做规划。把预测当成"冻结的真值"会导致**冻结机器人问题 (freezing robot)**——预测把所有空间填满，规划无路可走；解法是**交互式/条件预测**（预测以自车计划为条件）或博弈式联合规划。

## 3. 运动规划方法（采样、格搜、优化、MPC）

运动规划在选定行为意图后，求一条满足车辆动力学 $\dot{x}=f(x,u)$、无碰撞、且代价最小的轨迹。四大方法谱系：

### 3.1 采样法：RRT* 与 kinodynamic 变体

在状态空间随机采样、延展、回溯（详见 [运动规划](../../00_Robotics/02_Classical_Robotics/运动规划.md)）。RRT* 渐近最优；用于停车、掉头、非结构化场景（无车道线）。难点是**满足非完整约束**（车不能横移）——需用 kinodynamic RRT 或 Reeds-Shepp/Dubins 曲线做 STEER。

### 3.2 格搜：Lattice / Hybrid A*

结构化道路（高速、城市干道）的主力。

- **Lattice planner**：在 **Frenet 坐标系**（沿车道弧长 $s$、横向偏移 $d$）下，对终态 $(s,d,\dot s)$ 做规则**采样**，用**五次多项式**连接生成大量候选轨迹，逐条做碰撞/约束/代价评估取最优。Apollo EM planner、Werling 的横纵解耦法是代表。Frenet 框架的优势：把"贴着车道开"自动编码进坐标系，横纵向可解耦优化。
- **Hybrid A\***：在连续位姿上扩展、考虑最小转弯半径的离散搜索，泊车与窄道的标准选择。

### 3.3 优化法：直接把轨迹当变量

把轨迹离散为路点 $\xi=\{x_k\}$，最小化代价泛函并施加约束：

$$
\min_{\xi}\ \sum_k \big[ w_s\, c_{\text{safe}}(x_k) + w_c\, \|\ddot{\xi}_k\|^2 + w_e\, c_{\text{eff}}(x_k)\big]
\quad \text{s.t.}\ x_{k+1}=f(x_k,u_k),\ g(x_k)\le 0
$$

非凸碰撞约束用序列凸优化（TrajOpt 思想）线性化；自动驾驶常用 **iLQR / DDP**（沿轨迹二阶展开，warm-start 快）或 QP（凸化后毫秒级）。优势是轨迹平滑、约束显式；代价是依赖良好初值（由采样/格搜提供），易陷局部最优。

### 3.4 模型预测控制 (MPC)

把"滚动 horizon 优化 + 只执行首步"作为规划核心：每周期预测未来 $H$ 步、求最优控制序列、执行 $u_0$、下一周期用新观测重解。自动驾驶常用 horizon 3–5 s、频率 10–50 Hz，cost 含到参考线距离、加速度/jerk 平方、与他车间距平方，约束含速度/加速度上限与车道边界。Tube MPC / robust MPC 处理预测与模型的不确定性。

→ 完整推导、iLQR、线性/非线性/Tube 分类与稳定性见 **[MPC 笔记](../../00_Robotics/02_Classical_Robotics/MPC.md)**。MPC 既可作为运动规划器（输出轨迹），也可作为底层跟踪器（见 [车辆控制](../04_Control/车辆控制.md)），二者 horizon 与频率不同。

## 4. 代价函数设计（舒适 / 安全 / 效率 / 合规的多目标权衡）

规划质量 90% 取决于代价函数。典型加权和：

$$
J(\xi) = w_s\, J_{\text{safety}} + w_c\, J_{\text{comfort}} + w_e\, J_{\text{efficiency}} + w_r\, J_{\text{rules}}
$$

| 目标 | 常用项 | 工程含义 |
|---|---|---|
| **安全 safety** | 与他车/边界最小距离、碰撞 TTC、占据概率积分 | 通常做成硬约束或极高权重软约束 |
| **舒适 comfort** | $\int \|\ddot{\xi}\|^2$（加速度）、$\int \dddot{\xi}^2$（jerk）、横向加速度 | 直接影响乘员体感与晕车 |
| **效率 efficiency** | 行程时间、与期望速度偏差、变道收益 | 与安全/舒适天然冲突 |
| **合规 rules** | 限速、车道线、让行优先级、停止线 | 部分国家/地区可硬编码 |

**核心难点与对策：**

- **多目标不可公度**：把"米""秒""m/s³"加权求和缺乏物理意义。对策：分级（lexicographic，先满足安全再优化舒适）、归一化、或用约束代替软项。
- **权重调参爆炸**：手调 $w$ 在长尾场景顾此失彼。对策：**逆强化学习/逆最优控制 (IRL/IOC)** 从人类驾驶数据反推代价（ChauffeurNet、MaxEnt IRL）；或可学习代价（nuPlan 上的 learned planner）。
- **代价 ≠ 真实意图**：低代价轨迹可能"合法但反人性"（急刹避让一只塑料袋）。需场景化代价与风险分级。

## 5. 安全与可解释（可验证边界、责任敏感安全 RSS）

学习型规划器是黑箱，量产需要**可验证的安全边界**而非仅靠平均指标。

### 5.1 责任敏感安全 (RSS)

Mobileye 提出的形式化框架（Shalev-Shwartz et al. 2017）：用少量可解释规则给出**最小安全距离**等闭式边界，只要自车不违反这些边界，碰撞责任就不在自车。最具代表性的是纵向安全距离：

$$
d_{\min} = \Big[ v_r \rho + \tfrac{1}{2} a_{\max}\rho^2 + \tfrac{(v_r + \rho a_{\max})^2}{2 b_{\min}} - \tfrac{v_f^2}{2 b_{\max}} \Big]_+
$$

其中 $v_r,v_f$ 为后/前车速度，$\rho$ 为反应时间，$a_{\max}$ 为最大加速，$b_{\min},b_{\max}$ 为最小/最大合理减速。直觉：留出"自车以最坏反应时间加速、前车以最强减速"后仍能刹停的距离。RSS 给出横纵向、路口、遮挡等场景的规则，可作为规划器的**安全护栏 (safety shield)**——任何输出轨迹必须通过 RSS 检查，否则回退到保守动作。

### 5.2 其他可验证手段

- **可达性分析 / control barrier function (CBF)**：用前向可达集或 CBF 给出状态约束的形式保证，把安全编码为优化的硬约束。
- **safety shield / fallback**：学习型规划输出经规则/MPC 护栏过滤，违例则切到最小风险机动 (MRM)。
- **可解释代价**：把代价分解为命名项，便于事故复盘归因。

> 取舍：RSS 等基于最坏假设的边界**保守**（可能过度让行、被加塞），与效率冲突；研究方向是数据驱动地收紧最坏假设（如对他车减速能力做分布式估计）。

---

## 6. 对抗与长尾场景

规划的真正难度集中在分布尾部：

- **交互与博弈**：无保护左转、汇流、环岛——他车行为依赖自车动作，单向预测+规划会**冻结**或**过度保守**。对策：博弈式规划（Stackelberg/纳什）、条件预测、MPDM 多策略前向仿真。
- **遮挡与受限视野**：被大车挡住的横穿行人。对策：对**不可见区域**做最坏假设（phantom agent），或 POMDP 主动信息收集（缓行探头）。
- **长尾罕见事件**：逆行车、掉落货物、施工改道、特种车辆。监督学习样本稀缺，需仿真合成（[CARLA](../../00_Robotics/02_Classical_Robotics/运动规划.md) 类）+ 对抗场景生成。
- **对抗扰动**：感知层对抗补丁会污染预测进而误导规划；规划需对上游不确定性鲁棒（robust/risk-aware planning，对预测协方差敏感）。
- **闭环 vs 开环评估**：开环只比对单帧与人类轨迹的相似度（ADE/FDE），会**奖励模仿、掩盖复合误差**；必须做闭环仿真评估（nuPlan、CARLA leaderboard）才能暴露漂移与因果混淆。

## Common Pitfalls

- **盲目分层导致信息丢失**：行为层选错（如"该让却抢"），运动层无论怎么优化都救不回。需要把行为代价与运动代价对齐，或做联合/回退机制。
- **冻结机器人**：把预测当冻结真值、不考虑交互，密集场景里规划器找不到可行解而停滞。用条件/交互预测或博弈规划。
- **开环指标自欺**：仅用 ADE/FDE 或模仿损失调参，闭环一上路就漂移、复合误差爆炸。必须闭环评估（nuPlan/CARLA）。
- **代价权重过拟合 demo**：手调 $w$ 在录制片段上漂亮，长尾即崩。用 IRL/learned cost + 大规模闭环回归。
- **安全软约束被效率压过**：把碰撞做成可被高效率权重抵消的软项。安全应是硬约束或带 RSS 护栏的不可逾越边界。
- **Frenet 坐标在大曲率失真**：弯急、车道线缺失时 Frenet 投影病态，横纵解耦假设失效。需切回笛卡尔或 Hybrid A*。
- **MPC horizon/频率错配**：horizon 太短短视、太长模型误差累积；规划与跟踪两个 MPC 的 horizon/频率混淆。

## 前沿趋势

- **端到端规划**：UniAD（Hu 2023, CVPR Best Paper）把检测-跟踪-建图-预测-规划统一进一个可微管线，以规划为目标反向监督全栈；VAD（Jiang 2023, ICCV）用矢量化场景表征做高效端到端规划。弥合分层信息丢失，但牺牲可解释。
- **可学习代价 / 闭环模仿**：nuPlan（Caesar 2021）推动从"开环 ADE"转向"闭环规划评估"；learned planner + 规则护栏成为折中范式。
- **世界模型驱动规划**：MILE（Hu 2022, NeurIPS）、GAIA-1（Hu 2023）学可生成的驾驶世界模型，使规划器能在想象中 rollout（MPC 思想 × 生成模型），改善长尾覆盖。
- **博弈与交互式规划**：把规划显式建成多智能体博弈，缓解冻结/过度保守，逐步走向量产汇流/无保护左转。
- **形式化安全的数据化**：RSS、CBF、可达性与学习结合，用数据收紧最坏假设，在安全与效率间找更优前沿。
