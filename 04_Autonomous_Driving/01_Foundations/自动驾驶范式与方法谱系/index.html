    自动驾驶范式与方法谱系

    Autonomous Driving Paradigms — 把"感知-预测-规划-控制"这条流水线和"用什么方法做决策"这两件事分开看,自动驾驶的方法谱系就清楚了.

    难度：Intermediate 前置知识：模仿学习、MPC、强化学习在机器人中的应用 后续阅读：协变量偏移与闭环鸿沟、端到端驾驶

    1. 为什么需要一张地图

    ALVINN、PilotNet、Apollo、ChauffeurNet、UniAD、特斯拉 FSD……自动驾驶的"方法"名词多得让人眼花，初学者很容易把它们混成一锅粥：以为"端到端"就是"用神经网络"，以为"模仿学习"就是"端到端"，以为"模块化"就是"老旧的非学习方法"。

    这些混淆几乎都来自把两件正交的事情塞进了一个轴。理清自动驾驶只需要两根坐标轴：

    • 架构轴：从全解耦的模块化 pipeline，到一个网络到底的端到端 (E2E)。回答的是"系统由几块组成、信息怎么流"。
    • 学习/决策轴：从规则 + 最优控制，到模仿学习 (BC/IL)、强化学习 (RL)、世界模型 + MPC。回答的是"决策从哪来、用什么训"。

    本篇不深挖任何单个系统，而是把这张二维地图画出来，给每个格子点名代表作。读完你应该能一眼说出："UniAD 是端到端可微的模块化 + 模仿学习"而不是含糊地说"它是端到端的"。

    2. 第一根轴：系统架构（模块化 pipeline ↔ 端到端 E2E）

    2.1 模块化 pipeline：感知 → 预测 → 规划 → 控制

    经典自动驾驶栈把任务解耦成若干模块，模块之间用人为定义的中间表示衔接：

    • 感知：输出检测框、车道线、可行驶区域、占据栅格；
    • 预测：输出其他交通参与者的未来轨迹；
    • 规划：输出本车的目标轨迹/动作序列；
    • 控制：把轨迹变成方向盘/油门/刹车信号。

    优点是可解释、可调试、可分工——某个模块出错可以单独定位和替换，每个模块都能用各自的真值独立训练或验证。Apollo、Autoware 是典型代表。代价是中间表示是信息瓶颈：感知丢掉的信息后面再也拿不回来，且各模块各自优化不等于全局最优。

    2.2 端到端 E2E：传感器 → 控制，一个网络到底

    端到端把整条链路压成一个（可微的）网络，从原始传感器输入直接输出控制量（或轨迹）。优点是联合优化（梯度从控制端一路传回感知端，整个系统朝同一个目标对齐）、没有人为信息瓶颈。代价是黑箱难验证、数据饥渴，且——这是本板块的核心——它把模仿学习的协变量偏移问题暴露得最彻底。

    2.3 中间地带：端到端可微的模块化

    最常见的误解是把"端到端"等同于"没有模块"。不是的。 端到端的本质是梯度能从最终目标一路反传到最前端，而不是"网络里没有结构"。

    UniAD (CVPR 2023 Best Paper) 是典范：它保留了感知、预测、规划这些模块的语义（甚至每个模块都有自己的中间监督），但让它们串成一个可微整体，用最终的规划目标统一反传——所以它既是"模块化"（有清晰的模块边界）又是"端到端"（梯度贯通）。架构其实是一条连续光谱，而非二选一。

    Figure 1 — 架构光谱：左端是硬接口解耦的模块栈（可解释），右端是模块溶解的单一网络（联合优化），中间是 UniAD 式"保留模块语义但梯度贯通"的可微模块化。判别"是不是端到端"看的是那条贯穿全链的反传梯度箭头，而不是有没有盒子。

    3. 第二根轴：学习/决策范式

    同一种架构，决策可以用完全不同的方法产生：

    • 3.1 规则 + 最优控制：不"学"策略，靠工程化的有限状态机、代价函数和最优控制（MPC、LQR）求解。可证、可解释，但长尾场景靠人手写规则堆砌。
    • 3.2 模仿学习（IL / BC）：监督拟合专家的 (观测 → 动作) 映射，不需要奖励函数。最易上手，但有结构性的协变量偏移缺陷——本板块的核心难题。
    • 3.3 强化学习（RL）：用奖励在仿真/闭环中试错。多用于规划/控制层，或作为端到端策略的训练信号；真车上在线试错不安全，通常在仿真里做。
    • 3.4 世界模型 + MPC（model-based）：先学环境动力学（预测未来观测/状态），再在"想象"里 rollout 并规划。代表是 Wayve 的 MILE、GAIA-1，详见表征与世界模型。

    4. 正交性核心：BC ≠ E2E

    这是全篇最该记住的一点：

    BC（行为克隆）是一种学习方法——监督拟合专家的 (观测 → 动作)。 端到端 E2E 是一种架构——传感器到控制，一个网络到底。 二者位于不同的轴，可以任意组合。

    PilotNet = BC 且 E2E（对原始像素到方向盘做监督回归）；但你也可以对高层离散动作/轨迹做 BC 而完全不端到端（监督的是模块化中间表示，例如对 BEV 上的轨迹做克隆）。所以 BC ≠ E2E。

    4.1 四象限：每个格子都有居民

    Figure 2 — BC×E2E 二维象限：四个象限都有真实系统。右上（PilotNet）是 BC+E2E，右下（ChauffeurNet）是 BC 但模块化（对中间表示做克隆），左上（Roach）是 E2E 但用 RL，左下（Apollo）是模块化且非克隆。可见 BC 与 E2E 互不蕴含。

    4.2 三个常见混淆，一次说清

    1. "端到端" ≠ "没有模块"：UniAD 保留了感知/预测/规划模块，但梯度贯通，仍是端到端（见 §2.3）。
    2. "用了神经网络" ≠ "端到端"：模块化栈里每个模块都可以是 NN（感知用 CNN、预测用 Transformer），但接口是人为切断的，梯度不贯通，就不是端到端。
    3. "BC" ≠ "必须从像素出发"：可以对任意抽象层做 BC——对原始像素做（PilotNet），也可以对 BEV 中间表示、对高层离散动作或轨迹做（ChauffeurNet 的 mid-to-mid、Learning by Cheating 的特权-学生轨迹克隆）。

    5. 二维分类矩阵（核心交付物）

    把两根轴交叉，自动驾驶的代表系统各就各位：

    架构 ＼ 学习范式 规则 + 最优控制 模仿学习 (BC/IL) 强化学习 (RL) 世界模型 + MPC
    模块化 pipeline Apollo / Autoware 经典栈 Learning by Cheating（特权师生）、ChauffeurNet（mid-to-mid） 规划/控制层 RL（Apollo 内强化模块） 模块化预测 + MPC 规划
    端到端 E2E （罕见，纯优化的端到端少见） ALVINN、PilotNet、TransFuser、VAD 端到端 RL（CARLA 上的 Roach 等） MILE、GAIA-1（生成式世界模型）
    端到端可微·模块化 — UniAD（CVPR 2023 Best Paper）、VAD UniAD + RL 微调 可微世界模型 + 规划

    表里特意把"端到端可微·模块化"单列一行，因为 UniAD / VAD 既"端到端"（梯度贯通）又"保留模块语义"——再次印证架构是连续光谱而非二值。

    6. 代表系统年表

    系统 年份 / 出处 架构象限 学习范式
    ALVINN 1989, NeurIPS 端到端 E2E BC
    NVIDIA PilotNet 2016, arXiv 端到端 E2E BC（含三摄像头 recovery）
    ChauffeurNet 2018, arXiv / RSS 2019 模块化 (mid-to-mid) BC + 合成扰动
    Learning by Cheating 2019, CoRL 模块化 (特权师生) BC
    CARLA（仿真器/闭环榜） 2017, CoRL 评测平台 —
    Roach 2021, ICCV 端到端 E2E RL 教练 + IL 蒸馏
    TransFuser 2021 CVPR / 2022 TPAMI 端到端 E2E IL（多传感器融合）
    MILE 2022, NeurIPS (Wayve) 端到端 + 世界模型 IL（model-based）
    GAIA-1 2023, arXiv (Wayve) 生成式世界模型 自监督生成

    钟摆很清楚：1980s 的 E2E+BC（ALVINN）→ 2010s 工业界回到模块化栈（Apollo）→ 2020s 又用"端到端可微·模块化"（UniAD）与生成式世界模型把两者融合。

    维度 模块化+规则栈 模块化 IL 端到端 IL 端到端 RL 世界模型

    8. Common Pitfalls

    1. 把 BC 和 E2E 当同义词：本篇全篇要破除的头号误区（见 §4）。
    2. 以为端到端就没有中间表示：可微模块化（UniAD）反例。
    3. 以为模块化就一定不可微 / 一定不用 NN：模块化栈里每个模块都可以是 NN。
    4. 忽视"离线指标好 ≠ 闭环上路好"：纯靠开环 L2 选模型会被协变量偏移坑——详见协变量偏移与闭环鸿沟。
    5. 以为纯端到端必然更优：联合优化的代价是数据饥渴、难验证、协变量偏移最严重。

    9. 前沿趋势

    1. 端到端可微化：UniAD 之后，端到端但保留可解释模块语义成为主流方向。
    2. 生成式世界模型造长尾：GAIA-1 等用生成模型合成稀有危险场景，缓解数据与协变量偏移。
    3. VLA / 大模型进入驾驶：驾驶版视觉-语言-动作模型带来语言级可解释规划，详见VLA 模型。
    4. 闭环仿真评测成为标准：nuPlan、CARLA Leaderboard 把闭环成功率/违规分作为正式指标，详见数据集与闭环评测。
