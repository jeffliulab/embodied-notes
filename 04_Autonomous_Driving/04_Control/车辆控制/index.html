    车辆控制

    Vehicle Control — 自动驾驶栈的最后一公里：把规划层给出的参考轨迹（一条带速度的曲线）转成方向盘转角与油门/刹车，让车以厘米级精度、平顺且稳定地跟住它。核心矛盾是：车是一个非完整 (nonholonomic)、带执行延迟与轮胎非线性的二阶系统。

    难度：Intermediate 前置知识：机器人控制理论、MPC — 模型预测控制、规划与决策 后续阅读：端到端自动驾驶

    车辆控制接在运动规划之后：规划输出一条参考轨迹 \(\{(x_r,y_r,\theta_r,v_r,\kappa_r)\}\)，控制器负责实时把车的实际状态拉回到这条线上。它分两条相对独立又最终耦合的子任务：

    • 纵向控制 (longitudinal) —— 控制速度 / 车距，输出加速度 → 油门/刹车。
    • 横向控制 (lateral) —— 控制横向偏差 / 航向，输出前轮转角 → 方向盘。

    下面从最简单的运动学模型出发，逐步过渡到把两者统一在一个约束优化里求解的 MPC。

    1. 车辆动力学基础

    1.1 运动学自行车模型 (Kinematic Bicycle Model)

    把四轮车简化为前后两个等效轮（故名"自行车"）。设后轴中心为参考点 \((x,y)\)，车身航向角 \(\theta\)，前轮转角 \(\delta\)，轴距 \(L\)，后轴速度 \(v\)。无侧滑假设（轮子只沿自身朝向滚动，低速下成立）给出状态方程：

    \[ \dot{x} = v\cos\theta,\qquad \dot{y} = v\sin\theta,\qquad \dot{\theta} = \frac{v}{L}\tan\delta,\qquad \dot{v} = a \]

    几何直觉：前轮转角 \(\delta\) 决定瞬时转弯半径 \(R = L/\tan\delta\)，于是航向变化率 \(\dot\theta = v/R = (v/L)\tan\delta\)。这正是横向控制曲率公式的根基——控制器要算的"曲率"\(\kappa = \tan\delta / L\)，再反解出 \(\delta\)。

    如果参考点取在质心而非后轴，需引入侧偏角 \(\beta = \arctan\!\big(\tfrac{l_r}{l_f+l_r}\tan\delta\big)\)：

    \[ \dot{x} = v\cos(\theta+\beta),\quad \dot{y} = v\sin(\theta+\beta),\quad \dot{\theta} = \frac{v\cos\beta}{l_f+l_r}\tan\delta \]

    其中 \(l_f, l_r\) 为质心到前/后轴距离。运动学模型只在低速（< ~5 m/s）且小转角时准确，因为它假设轮胎不打滑。

    1.2 动力学模型与轮胎侧偏

    高速过弯时轮胎会产生侧偏角 (slip angle) \(\alpha\)——轮胎实际速度方向与轮胎朝向的夹角。线性轮胎模型下侧向力正比于侧偏角：

    \[ F_{y,f} = C_{\alpha f}\,\alpha_f,\qquad F_{y,r} = C_{\alpha r}\,\alpha_r \]

    \(C_\alpha\) 为侧偏刚度 (cornering stiffness)。以横向速度 \(v_y\) 与横摆角速度 \(r=\dot\theta\) 为状态的线性单轨动力学模型：

    \[ m(\dot{v}_y + v_x r) = F_{y,f} + F_{y,r},\qquad I_z \dot{r} = l_f F_{y,f} - l_r F_{y,r} \]

    其中侧偏角近似 \(\alpha_f = \delta - \tfrac{v_y + l_f r}{v_x}\)、\(\alpha_r = -\tfrac{v_y - l_r r}{v_x}\)。要点：

    • 侧偏刚度有限 → 高速时实际转弯半径大于运动学预测（"转向不足"），运动学模型会让控制器偏置。
    • 轮胎力饱和：真实轮胎是非线性的（Pacejka "魔术公式"），接近附着极限时 \(F_y\) 不再随 \(\alpha\) 增长 → 失稳。
    • 工程上：低速泊车用运动学模型，高速 / 赛道用动力学模型；MPC 里常按速度切换或用动力学模型全程覆盖。

    2. 纵向控制

    目标：跟踪参考速度 \(v_r\)（巡航）或参考车距 \(d_r\)（跟车 / ACC）。控制量是期望加速度 \(a\)，再经逆纵向模型（发动机/电机 map + 刹车）转成油门/刹车开度。

    2.1 PID 速度控制

    最朴素也最常用：

    \[ a = K_p(v_r - v) + K_i\!\int(v_r - v)\,dt + K_d\frac{d}{dt}(v_r - v) \]

    积分项消除上坡 / 风阻导致的稳态误差；微分项抑制超调（但对速度噪声敏感，常用低通滤波）。前馈很关键：参考已知加速度 \(a_r\) 时直接 \(a = a_r + \text{PID}\)，把 PID 留给残差，跟踪显著更准。

    ACC 跟车则用双环：外环按车距误差 \(d - d_r\)（含时距 headway \(d_r = d_0 + \tau_h v\)）算期望速度，内环 PID 跟速度。

    2.2 纵向 MPC

    把速度规划写成优化（尤其在限速、坡度、前车切入等约束多时）：

    \[ \min_{a_{0:H-1}}\ \sum_{k=0}^{H-1}\!\Big[q\,(v_k - v_{r,k})^2 + r\,a_k^2 + r_\Delta\,(a_k-a_{k-1})^2\Big] \quad\text{s.t.}\quad \begin{cases} s_{k+1}=s_k+v_k\Delta t\\ v_{k+1}=v_k+a_k\Delta t\\ \underline{a}\le a_k\le\bar a,\ \ |a_k-a_{k-1}|\le \Delta a_{\max}\\ s_k \le s_{\text{lead},k}-d_{\text{safe}} \end{cases} \]

    加加速度项 \((a_k-a_{k-1})^2\) 直接优化舒适性 (jerk)；前车安全距离作为硬约束自然处理跟车切入。这是凸 QP，几 ms 即可解。

    3. 横向控制

    3.1 Pure Pursuit（纯追踪）

    思想极简：在参考路径上选一个前视点 (lookahead point)，距车后轴 \(L_d\)（前视距离），令车走一段圆弧正好穿过该点。下图给出几何推导。

    Figure 1 — Pure Pursuit 的几何：从后轴 (x,y) 到前视点拉一条弦 L_d，弦与车身航向夹角 α，追踪圆弧（橙）半径 R，圆心 (ICR) 在车身横轴上。由等腰三角关系 L_d/sin(2α)=2R 得曲率 κ=2 sin α/L_d。

    曲率推导：设追踪圆过当前后轴点与前视点。圆心在车身横轴（垂直于航向）上，到两点距离均为 \(R\)。前视点相对车身的弦长 \(L_d\)，弦与航向夹角 \(\alpha\)。由圆的几何（弦 \(L_d\) 对应圆心角 \(2\alpha\)）：

    \[ \frac{L_d}{\sin(2\alpha)} = 2R \;\Longrightarrow\; \frac{L_d}{2\sin\alpha\cos\alpha}=2R \]

    注意车横向偏移 \(e_y = L_d\sin\alpha\)，纵向 \(L_d\cos\alpha\)，可直接得到更常用形式：

    \[ \boxed{\ \kappa = \frac{1}{R} = \frac{2\sin\alpha}{L_d}\ } \qquad\Longrightarrow\qquad \delta = \arctan(\kappa L) = \arctan\!\Big(\frac{2L\sin\alpha}{L_d}\Big) \]

    调参核心是前视距离 \(L_d\)，常取速度相关 \(L_d = k_v v + L_0\)：低速短前视（精准、贴线）、高速长前视（平顺、不振荡）。\(L_d\) 太短 → 高频振荡；太长 → 转弯切内 / 抄近道 (corner cutting)。Pure Pursuit 只用几何、无航向误差项，对高曲率路径有稳态偏差。

    3.2 Stanley 控制

    斯坦福 DARPA 夺冠车的横向控制器，以前轴为参考点，同时纠正横向误差 \(e\) 与航向误差 \(\theta_e\)：

    \[ \delta = \theta_e + \arctan\!\Big(\frac{k\,e}{v + k_s}\Big) \]
    • 第一项 \(\theta_e = \theta_r - \theta\) 让车头对齐路径切线方向；
    • 第二项是非线性反馈：横向误差大时转角接近 \(\pm\pi/2\)（直冲路径），误差小时线性收敛；分母 \(v\) 让高速时增益自动减小（避免高速发抖），\(k_s\) 防止低速除零。

    可证在自行车模型下 \(\dot e \approx -\,kv\cdot e/(v) = -k e\) 指数收敛。Stanley 比 Pure Pursuit 更贴线（无稳态横向偏差），但对横向误差测量噪声更敏感，且不显式考虑曲率前馈。

    3.3 LQR 横向控制

    把横向动力学线性化为 \(\dot{\mathbf{x}} = A\mathbf{x} + B\delta\)，状态取 \(\mathbf{x}=[e_y,\ \dot e_y,\ e_\theta,\ \dot e_\theta]^\top\)（横向误差、其变化率、航向误差、其变化率）。设计二次代价：

    \[ J = \int_0^\infty (\mathbf{x}^\top Q\,\mathbf{x} + R\,\delta^2)\,dt \]

    解连续 Riccati 方程 \(A^\top P + PA - PBR^{-1}B^\top P + Q = 0\)，得最优反馈 \(\delta = -K\mathbf{x}\)，\(K = R^{-1}B^\top P\)。实践中再叠加曲率前馈 \(\delta_{ff} = \arctan(\kappa_r L)\) 消除弯道稳态误差。LQR 显式权衡跟踪精度 (\(Q\)) 与控制平顺 (\(R\))，比手调 Pure Pursuit/Stanley 更系统，但仍是无约束、单点线性化。详见 机器人控制理论。

    3.4 横向 MPC

    把横向误差动力学放进滚动优化，显式约束转角与转角速率：

    \[ \min_{\delta_{0:H-1}}\sum_k \big(e_{y,k}^2 q_1 + e_{\theta,k}^2 q_2 + \delta_k^2 r + \Delta\delta_k^2 r_\Delta\big) \quad\text{s.t.}\quad |\delta_k|\le\bar\delta,\ |\Delta\delta_k|\le\Delta\bar\delta \]

    MPC 天然处理执行器物理上限与曲率前瞻（能"看到"未来的弯并提前打方向），是当前量产高阶辅助驾驶横向控制的主流。

    4. 横纵耦合与 MPC 统一

    前三节把横纵向解耦，但二者在物理上耦合：高速过弯时纵向减速能扩大轮胎侧向力裕度（"弯前刹车"）；轮胎附着力是横纵共享的有限资源（摩擦圆 \(\sqrt{F_x^2+F_y^2}\le\mu m g\)）。统一做法是用一个非线性 MPC 直接在自行车（或动力学）模型上做轨迹跟踪：

    \[ \min_{\mathbf{u}_{0:H-1}}\ \sum_{k=0}^{H-1}\!\Big[\|\mathbf{p}_k - \mathbf{p}_{r,k}\|_Q^2 + \|\mathbf{u}_k\|_R^2 + \|\Delta\mathbf{u}_k\|_{R_\Delta}^2\Big] + \|\mathbf{p}_H-\mathbf{p}_{r,H}\|_{Q_f}^2 \]
    \[ \text{s.t.}\quad \mathbf{x}_{k+1}=f(\mathbf{x}_k,\mathbf{u}_k),\quad \mathbf{u}_k=[\delta_k, a_k],\quad \underline{\mathbf{u}}\le\mathbf{u}_k\le\bar{\mathbf{u}},\ \ \sqrt{a_{x}^2+a_{y}^2}\le\mu g,\ \ \mathbf{x}_0=\mathbf{x}_{\text{meas}} \]

    其中 \(f\) 是 §1 的离散化自行车/动力学模型，\(\mathbf{p}=(x,y)\)，摩擦圆约束把横纵耦合编码进可行域。这正是 MPC 的标准应用——滚动时域、只执行第一步、每步重解。用 Frenet 坐标（沿路 \(s\)、横向 \(e_y\)）可把"跟轨迹"写得更自然，并把横纵权重在曲线坐标下分离。规划层（规划与决策）给的就是这条参考 \(\mathbf{p}_{r}\) 与 \(v_r\)。

    5. 执行与稳定性

    控制律在纸上漂亮，落地时被三件事反复折磨：

    5.1 延迟 (latency)

    从相机/雷达到转角生效有 100–300 ms 的链路延迟（感知、规划、CAN 总线、转向执行器机械响应）。延迟会让闭环相位裕度下降乃至失稳，尤其高速。对策：

    • Smith 预估器 / 状态外推：用模型把当前状态前推一个延迟周期，对"未来状态"控制。
    • MPC 内建延迟模型：把执行器一阶滞后 \(\dot\delta = (\delta_{cmd}-\delta)/\tau\) 写进 \(f\)。

    5.2 饱和 (saturation)

    物理上限：\(|\delta|\le \delta_{\max}\)（约 ±30–40°转向轮）、\(|\dot\delta|\le\dot\delta_{\max}\)（方向盘转速）、加速度/加加速度上限。无约束控制器（PID/LQR）算出的指令一旦超限被硬截断，闭环行为偏离设计。MPC 把这些作为硬约束天然处理；PID/LQR 则需显式 clamp + 抗饱和。

    5.3 抗积分饱和 (anti-windup)

    PID 的积分项在执行器饱和时仍持续累积（指令已顶到上限但误差不消），饱和解除后产生巨大超调。标准对策：

    • Clamping / conditional integration：饱和时冻结积分。
    • Back-calculation：把"指令 − 实际饱和输出"反馈回积分器 \(\dot I = K_i e + K_t(u_{sat}-u)\)。

    外加：转角速率限制 (rate limiting) 保护机械、平滑滤波抑制抖振、以及当 MPC 求不出可行解时的 fallback（降级到 Pure Pursuit + 安全减速）。

    方法对比

    方法 用什么误差 约束处理 前瞻 计算量 最适场景
    Pure Pursuit 前视点几何（横向） 无 隐式（靠 \(L_d\)） 极低 低速、低曲率、算力受限；泊车/园区
    Stanley 横向 + 航向误差 无 无 极低 中低速贴线跟踪（DARPA 越野）
    LQR 4 维误差状态 无（需 clamp） 单点线性化 低 速度稳定、模型已知的结构化道路
    MPC 全状态 + 轨迹 硬约束（转角/加速度/摩擦圆） 显式 H 步 高（QP/NLP） 高速、强约束、横纵耦合；量产高阶辅助
    学习/E2E 像素→控制 隐式 学到的 推理中等 复杂城区、难建模场景（研究/前沿）

    经验法则：算力紧 + 低速 → Pure Pursuit/Stanley；要稳态精度 + 已知模型 → LQR + 曲率前馈；有执行器约束 / 高速 / 横纵耦合 → MPC。

    Common Pitfalls

    • 运动学模型跑高速：低速假设的无侧滑模型在高速大横向加速度下系统性偏置，转弯不足。高速务必上动力学模型或在线辨识侧偏刚度。
    • Pure Pursuit 前视距离写死：固定 \(L_d\) 在低速振荡、高速抄近道。必须 \(L_d = k_v v + L_0\) 随速度自适应。
    • 忽略执行延迟：在仿真里完美的 PID/LQR 上车后高速发抖——根因常是 150–300 ms 链路延迟未补偿。
    • PID 无抗饱和：上坡满油门 / 急转方向到底后，积分项 windup 导致松开后剧烈超调。
    • 横纵彻底解耦：弯道里横向 MPC 要急打方向、纵向却没减速，超出摩擦圆而打滑。共享附着力需联合约束。
    • MPC 无可行解时无 fallback：约束太紧或初值病态导致 solver 失败，必须有降级控制器与安全减速兜底，否则方向盘"卡死"。
    • Stanley 在低速除零：分母 \(v+k_s\) 的 \(k_s\) 不能省，否则停车附近转角发散。
    • 代价权重未归一化：\(Q,R\) 量纲不一致（米 vs 弧度）导致调参玄学，应按物理尺度先归一化。

    前沿趋势

    • 可微分 / 学习增强 MPC：把 MPC 作为可微层嵌进网络端到端学代价权重与动力学残差，兼顾约束保证与数据自适应。
    • 端到端控制：从 ALVINN(1989)、PilotNet(2016) 到 UniAD(2023)、VAD(2023) 直接从传感器到轨迹/控制；详见 端到端自动驾驶。控制层逐渐被"轨迹规划 + 轻量跟踪器"或全栈神经网络吸收。
    • 世界模型驱动控制：MILE(2022)、GAIA-1(2023) 学车载世界模型，在想象中 rollout 做 model-based 控制 / 评估，是 MPC 思想的神经化。
    • 接触/极限工况控制：赛车级 MPC（漂移、附着极限）与轮胎力实时估计，把摩擦圆约束推到边界。
    • 安全过滤器 (safety filter)：用控制屏障函数 (CBF) / 可达集在学习策略外套一层最小干预的安全层，兼顾性能与可证安全。
