    协变量偏移与闭环鸿沟

    Covariate Shift & the Closed-Loop Gap — 行为克隆为什么在驾驶里会"越开越偏直至雪崩",以及业界用三摄像头、合成扰动、闭环训练怎么把它摁住.

    难度：Intermediate-Advanced 前置知识：模仿学习（含 Ross & Bagnell \(O(\epsilon T^2)\) 定理与 DAgger）、自动驾驶范式与方法谱系 后续阅读：端到端驾驶、数据集与闭环评测、强化学习在机器人中的应用

    1. 术语与定义

    1.1 协变量偏移 / 输入分布偏移

    协变量偏移（covariate shift），中文也叫输入分布偏移，指的是：

    \[ p_{\text{train}}(x) \neq p_{\text{test}}(x), \quad \text{而条件关系 } p(y \mid x) \text{ 不变} \]

    即训练时见到的输入分布与部署时的输入分布不一致，但"给定输入、正确答案是什么"这件事没变。模型在训练分布上学得再好，一旦部署时输入落到训练里罕见/没见过的区域，预测就不可靠。

    1.2 与近邻概念的区分

    偏移类型 变的是什么 驾驶里的例子
    协变量偏移 covariate shift 输入边际 \(p(x)\) 变，\(p(y\mid x)\) 不变 训练在晴天市区，部署到雨夜郊区
    标签偏移 label / prior shift 标签边际 \(p(y)\) 变 训练集里"直行"占 90%，部署在弯道密集区
    概念漂移 concept drift \(p(y\mid x)\) 本身随时间变 交规改了、车道线重画

    本篇只谈第一种。关键在于：在模仿学习/闭环控制里，这个 \(p(x)\) 的偏移不是外界强加的，而是策略自己造出来的。

    2. 两层视角：通用 ML 的偶发 vs IL/闭环的必然

    2.1 通用监督学习里：外因、偶发

    普通 ML 假设数据独立同分布。协变量偏移在这里是外因偶发的——采样偏差、地域/季节变化、传感器更换。数据集是固定的，模型不会改变自己将来会见到什么。处理手段是重要性加权、域适应等"被动补偿"。

    2.2 IL / 闭环控制里：内因、结构性必然

    模仿学习的策略一旦上车闭环运行，情况质变：策略当前的输出决定了下一时刻它会看到什么状态。于是策略诱导出自己的状态访问分布 \(d_\pi\)，它与专家的 \(d_{\pi^*}\) 必然不同：

    \[ d_\pi \neq d_{\pi^*} \]

    这就是闭环（策略影响自己的输入分布）与开环（数据集固定不动）的根本差异。在普通监督学习里协变量偏移是"可能发生"，在 IL 闭环里是"一定发生"——只要策略不是和专家逐比特相同，它就会把自己带到专家没演示过的状态。

    2.3 误差界（引用，不重证）

    模仿学习笔记已给出 Ross & Bagnell 的经典结论：若行为克隆每步出错概率为 \(\epsilon\)，则 \(T\) 步轨迹的总损失上界是 \(J(\pi_\theta)-J(\pi^*)\le O(\epsilon T^2)\)（二次增长），而 DAgger 通过在策略自身访问的状态上查询专家，把它降到 \(O(\epsilon T)\)（线性）。本篇不重复推导，直接用这个结论解释驾驶。

    2.4 为何"驾驶里 ε 极小也致命"

    二次界只是统计期望。在自动驾驶里有三重放大，使得即便每步误差 \(\epsilon\) 极小也可能致命：

    • 非线性吸收态 → 正反馈雪崩：车辆动力学使一次小偏离不是"线性累加的误差"，而是把状态推向训练分布外的"吸收区"（压线、骑上路肩、切入对向车道）。一旦进入策略从未见过的状态，输出近乎随机 → 偏得更远 → 状态更陌生，构成正反馈，误差指数级离开专家流形（见 Figure 1）。
    • 安全的非对称代价：监督损失把所有误差等权（偏 0.5 m 和偏 2 m 的 MSE 差不多），但真实世界的代价是阶跃式的——偏 0.5 m 还在车道内，偏 2 m 就出界/碰撞。所以"期望误差 \(\epsilon\) 很小"会严重低估尾部风险。
    • 时间尺度：\(\epsilon T^2\) 里的 \(T\) 在高速场景对应很大的物理位移，长程任务下 \(T\) 绝不可忽略。

    Figure 1 — 协变量偏移在闭环里的雪崩：策略轨迹（橙）起初贴着专家窄带（蓝，高数据密度）行驶，但每步小误差把它向外推；一旦冲出训练分布进入 OOD 区（灰），输出近乎随机，误差从线性变为正反馈式指数发散。这正是 $d_\pi\neq d_{\pi^*}$ 在物理闭环里的几何图像。

    3. 开环评估 vs 闭环评估的鸿沟

    协变量偏移最隐蔽的后果，是它让离线评估系统性地高估模型的真实表现。

    3.1 开环（离线回放）评估

    把模型放在专家录制的轨迹上，逐帧比对它预测的动作与专家动作（指标如 L2 位移误差、动作分类准确率）。注意：状态推进始终由专家驱动，模型从不真正开车——它永远待在 \(d_{\pi^*}\) 上，协变量偏移根本没机会发生。结果是系统性乐观。

    3.2 闭环（仿真/上路）评估

    让模型实际驱动车辆，下一时刻的状态由它自己的动作演化而来。这才暴露 \(d_\pi\neq d_{\pi^*}\) 与误差累积，指标是成功率、碰撞率、违规分、接管次数。

    3.3 为什么离线指标好 ≠ 上路好

    Codevilla 等 (2018) 的关键实证发现：开环 L2 误差与闭环驾驶成功率只有弱相关，甚至会出现反相关——一个开环 L2 更低的模型，上路成功率反而更差。原因正是开环评估把模型锁在专家分布上，量不到它脱离分布后会怎样。只看离线指标选模型，会被协变量偏移系统性地误导。

    Figure 2 — 开环 vs 闭环评估鸿沟：开环（上，绿）让模型在固定专家轨迹上"对答案"，状态由专家推进，永不离分布，L2 看起来很低；闭环（下，橙）让模型自己开，动作反馈进状态更新（自环箭头），误差累积导致偏离与碰撞。两者的差距（GAP）正是协变量偏移的度量。

    3.4 闭环 leaderboard

    诊断协变量偏移的唯一可信尺子是闭环评测：

    • CARLA Leaderboard：在 CARLA 仿真城市里跑预设路线，计驾驶分（行驶完成度 × 违规惩罚：闯红灯、碰撞、压线各扣分）。
    • nuPlan：首个大规模真实数据闭环规划基准，区分非反应式（背景交通按 log 回放）与反应式（背景交通对本车反应）两种闭环，更贴近真实交互。

    详见数据集与闭环评测。

    4. 解决方法谱系

    所有解法本质上都是想办法让训练分布覆盖到策略实际会访问的 \(d_\pi\)，区别只在"用什么方式逼近 \(d_\pi\)、要不要在线专家、要不要仿真"。

    Figure 3 — 两条人造 recovery 数据路线：左为 NVIDIA 三摄像头视角合成（用左右偏置摄像头制造"偏离"状态并打上纠回标签，空间偏置）；右为 ChauffeurNet 主动扰动专家轨迹并以辅助损失（碰撞/离路）惩罚危险状态（时序偏置）。两者都在不查询在线专家的前提下，把策略将会遇到的 $d_\pi$ 提前塞进训练分布。

    4.1 DAgger 及其驾驶变体

    核心思想是在策略自己访问的状态上查询专家标签，把训练分布主动拉向 \(d_\pi\)（框架与 \(O(\epsilon T)\) 保证见模仿学习）。驾驶落地的难点是在线查询专家昂贵且危险（要人类司机实时盯着纠正）。变体：SafeDAgger（只在策略不确定时才查询专家，省标注）、HG-DAgger（人在环交互式接管纠正）、或用一个规则/特权专家替代人类专家。

    4.2 NVIDIA 三摄像头 / 视角合成做 recovery

    PilotNet 在车上装左/中/右三个摄像头：中间摄像头配专家转向作正常样本，左右偏置摄像头配上"应当向右/左纠正"的转向标签——人为制造大量"偏离-纠回"样本（再辅以几何视角合成插值更多偏移量）。用数据而非在线查询近似了 DAgger 的效果，是工程上最朴素有效的一招（见 Figure 3 左）。

    4.3 ChauffeurNet 的合成扰动 + 辅助损失

    ChauffeurNet 在 mid-to-mid 表示上训练，关键两招：① 训练时主动扰动专家轨迹（制造离路、偏移、急刹），逼网络学会从坏状态纠回；② 加辅助损失（碰撞、偏离道路、保持在道、几何一致），显式惩罚危险状态。相当于在训练里"预演灾难"，是对 §4.2 思路的进一步系统化（见 Figure 3 右）。

    4.4 噪声注入 / DART

    DART (Laskey et al., 2017) 在收集演示时向专家动作注入噪声，使专家自然地访问并演示"偏离-纠回"，从而扩大演示覆盖的状态分布。它在采集阶段近似 DAgger，但无需在线专家，比 DAgger 更易落地。

    4.5 闭环训练 + RL 微调

    让策略在仿真闭环里自己 rollout，用 RL/奖励（或在线 DAgger）直接修正分布外行为——把"训练分布 = 部署分布"变成现实。这正对应范式谱系里"端到端 RL"象限（如 Roach 用 RL 教练在 CARLA 闭环里训练再蒸馏给 IL 学生）。

    4.6 数据增广与长尾场景挖掘

    主动挖掘长尾/接管/disengagement 场景定向补数据：影子模式（量产车后台跑模型、与人类司机分歧时记录）、难例回灌、对抗场景生成。本质是定向扩充 \(d_\pi\) 中的高风险尾部。

    4.7 世界模型生成长尾

    用生成式世界模型（GAIA-1 等）合成稀有危险场景，既做 recovery 训练数据，又做闭环评测环境，绕开真实世界采集这些场景的风险与成本。详见表征与世界模型。

    5. 经典案例串讲：一部"对抗协变量偏移"的进化史

    • ALVINN (1989)：朴素像素→转向 BC，没有任何 recovery 机制，偏离即失效——协变量偏移的原始受害者。
    • PilotNet (2016)：同样是 E2E+BC，但首次用三摄像头/视角合成系统性注入 recovery 数据（§4.2），让车能从偏离中纠回。
    • ChauffeurNet (2018)：搬到 mid-to-mid 模块化表示，用合成扰动 + 辅助损失在训练里"预演灾难"（§4.3），并显式罚危险状态。

    一句话收尾：从"无视"→"造数据"→"造灾难 + 罚危险"→"闭环训练 / 世界模型造长尾"，是同一个协变量偏移问题被层层逼近的历史。

    6. 方法对比

    见 §4 末尾小结表。一句话排序：最易落地 三摄像头/DART/扰动（无需在线专家、无需仿真）→ 最贴近本质 闭环训练/RL 微调（直接在 \(d_\pi\) 上训，但要仿真）→ 覆盖长尾最强 世界模型生成（但训练/评测成本高）。

    7. Common Pitfalls

    1. 只看开环 L2 就上车：协变量偏移会让离线指标系统性乐观（§3.3），上路前必须闭环评测。
    2. 以为加大 BC 数据量就能根治：扩大 \(d_{\pi^*}\) 上的数据治标不治本，因为问题出在 \(d_\pi\) 上策略没见过的状态；DAgger/recovery 才是对症。
    3. 把 recovery 数据当普通增广：随机增广若不覆盖真实失败模式（压线、切对向、急刹），对协变量偏移帮助有限。
    4. DAgger 直接上真车：在线让一个未成熟策略开车采数据有安全风险，需 SafeDAgger/人在环兜底。
    5. 闭环评测用非反应式背景交通：背景车不对本车反应会让分数虚高，真实交互（反应式，如 nuPlan 反应式赛道）才暴露问题。

    8. 前沿趋势

    1. 闭环可微仿真 + RL 微调成为端到端训练标配，从源头消除分布鸿沟。
    2. 生成式世界模型造长尾（GAIA 系）：合成稀有危险场景做闭环训练与评测。
    3. 大模型 / VLA 带来语言级可解释纠错：用语言推理识别并解释"我正在偏离"。
    4. 真实-仿真混合闭环评测：nuPlan / CARLA 之后，把真实日志重演与仿真反应式交通结合，逼近真实分布。
