# 自动驾驶

自动驾驶是**协变量偏移（covariate shift）、模仿学习与闭环控制**在真实物理世界中最尖锐的交汇点：策略在开环数据上训练，却必须在自己生成的状态分布里闭环运行；一帧的微小误差会沿着车辆动力学被积分、放大，最终演变成驶离车道的灾难性轨迹。它把感知、预测、规划、控制四个传统模块，连同近年崛起的端到端学习，压缩进同一个 10–100 ms 的延迟预算和同一条安全冗余链路里。

本章从两个互补的角度组织内容。**「方法谱系」**回答“有哪些范式、它们在归纳偏置上如何取舍”——从经典的模块化流水线（手工接口、可解释、可验证）到端到端可微分系统（数据驱动、联合优化、但接口黑箱），以及二者之间的混合形态。**「完整流水线」**则沿着信息从传感器到方向盘的真实流向，逐站拆解感知与定位、运动预测、规划决策、车辆控制，最后落到端到端驾驶与闭环评测——并始终把“为什么开环指标好、闭环却翻车”这条暗线贯穿始终。

## 推荐阅读顺序

1. 先读 **[自动驾驶综述](01_Foundations/自动驾驶综述.md)**，在脑中建立整个领域的地图——分级标准、模块边界、传感器配置与核心难题。
2. 接着 **[自动驾驶范式与方法谱系](01_Foundations/自动驾驶范式与方法谱系.md)**，理解“端到端 ≠ 模仿学习”这一关键区分，把后续每一篇都挂到正确的坐标上。
3. 沿流水线纵向打通：**[感知与定位](02_Perception/感知与定位.md)** → **[运动预测](03_Prediction_Planning/运动预测.md)** → **[规划与决策](03_Prediction_Planning/规划与决策.md)** → **[车辆控制](04_Control/车辆控制.md)**，看清信息如何从原始传感器流到方向盘扭矩。
4. 再读 **[端到端驾驶](05_Learning_E2E/端到端驾驶.md)**，理解可微分流水线如何把上述四步合并联合优化。
5. 然后是全章的“暗线” **[协变量偏移与闭环鸿沟](05_Learning_E2E/协变量偏移与闭环鸿沟.md)**——这是理解一切失败模式的钥匙。
6. 最后用 **[数据集与闭环评测](06_Data_Eval/数据集与闭环评测.md)** 收尾，学会用正确的尺子衡量进步，避免被开环指标误导。
